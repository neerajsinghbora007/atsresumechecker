🟢 GA4 Fundamentals

Q1. What is GA4 and why did Google replace Universal Analytics?
Beginner▾

Google Analytics 4 (GA4) is Google's current analytics platform that replaced Universal Analytics (UA) in July 2023. Key reasons for the replacement:
1. Privacy regulations (GDPR, iOS 14+) — UA relied heavily on third-party cookies. GA4 uses event-based tracking and machine learning to fill data gaps when cookies are unavailable.
2. Cross-platform tracking — GA4 unifies web and app data in one property
3. Event-based model — More flexible and accurate than UA's session-based model
4. BigQuery integration — GA4 allows free export to BigQuery for advanced analysis

🇮🇳 India Example: India Example: Nykaa tracks users across their website and mobile app. With UA, they had two separate properties — one for web, one for app. In GA4, one property captures the full journey: user first opens app (app event), browses products (view_item event), abandons cart on web (cart_abandonment event), and finally purchases on app after seeing a retargeting ad (purchase event). The complete funnel, connected.
Q2. What is the difference between events, parameters, and user properties in GA4?
Beginner▾

Events — Actions that users take on your website/app. Everything in GA4 is an event. Auto-collected: page_view, session_start. Enhanced: scroll, click, file_download. Custom: form_submit, video_play.
Parameters — Additional context about events. Every event can have parameters. Example: purchase event has parameters: currency, value, transaction_id, items.
User Properties — Attributes of users that persist across sessions. Examples: membership_level, preferred_language, user_tier.

🇮🇳 India Example: India Example: BigBasket tracking:
Event: add_to_cart
Parameters: item_name='Amul Butter 500g', item_category='Dairy', price=145, currency='INR'
User Property: customer_tier='Premium', city='Mumbai'
This gives BigBasket complete context: who added what to cart, at what price, and who the customer is — enabling precise personalization.
Q3. What is the difference between dimensions and metrics in GA4?
Beginner▾

Dimensions — Qualitative attributes that describe data (the 'what' and 'who'). Examples: Country, Device category, Page title, Campaign name, Event name.
Metrics — Quantitative measurements (the 'how many'). Examples: Sessions, Users, Conversions, Revenue, Bounce rate, Engagement rate.
Simple rule: Dimensions describe, metrics measure. You combine them: [Dimension] had [Metric] — 'Delhi (dimension) had 5,000 sessions (metric).'

🇮🇳 India Example: India Example: GA4 report for MakeMyTrip:
Dimension: Landing page = /flights/delhi-to-goa
Metric: Sessions = 45,000, Conversions = 450, Conversion rate = 1%
Insight: High traffic, average conversion. Action: A/B test the page to improve conversion rate.

🟡 Advanced GA4 Concepts

Q4. What is Consent Mode in GA4 and why is it important?
Intermediate▾

Consent Mode is a Google framework that adjusts how GA4 collects data based on user cookie consent. When users decline cookies (common in EU under GDPR, and increasingly in India too):
Without Consent Mode: No data collected for non-consenting users → data gaps → wrong insights
With Consent Mode v2: GA4 uses machine learning to model conversion behavior for non-consenting users, filling data gaps without collecting personal data
Two key parameters: analytics_storage (analytics consent), ad_storage (advertising consent)

🇮🇳 India Example: India Example: A European user visits an Indian brand's website and declines cookies. Without Consent Mode, this visit is completely invisible to GA4. With Consent Mode v2, GA4 models this user's likely behavior based on similar users who did consent — filling the gap. The brand sees 15% more accurate conversion data. As India moves toward DPDP (Digital Personal Data Protection Act), Consent Mode will become critical for Indian businesses too.
Q5. What are Explorations in GA4 and when do you use them?
Intermediate▾

Explorations are GA4's advanced analysis workspace for custom reports beyond standard dashboards. Types:
Free Form — Custom pivot table with any dimensions/metrics
Funnel Exploration — Visualize conversion funnels, identify drop-off points
Path Exploration — See the actual paths users take through your site
Segment Overlap — Compare behavior of different user segments
Cohort Analysis — Group users by acquisition date and analyze retention
User Lifetime — Long-term value and behavior of users

🇮🇳 India Example: India Example: Zomato uses Funnel Exploration to analyze their order funnel: Home page → Restaurant listing → Menu view → Cart → Checkout → Order confirmed. They find 45% of users drop at checkout. Drilling deeper in GA4 Exploration, they see this drop is 70% higher on Android devices — a specific bug on the Android checkout page. One exploration report → one bug fix → significant revenue recovery.
